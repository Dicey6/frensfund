OBJECTIVE

A user should be able to:

* Select a challenge.
* Create an order.
* View the payment page.
* Send SOL.
* Have payment verified automatically.
* Have their Demo Challenge activated.
* Have their dashboard update automatically.

No manual intervention.

ORDER CREATION

When a user selects a challenge:

Create a pending order.

Store:

* User ID
* Challenge Plan
* Purchase Price (USD)
* Current SOL Price
* Required SOL
* Treasury Wallet
* Wallet Address the user will pay from
* Order Status
* Created Time
* Expiry Time

Status should initially be:

Pending Payment

⸻

PAYMENT PAGE

Display:

* Challenge Name
* Purchase Price (USD)
* Live SOL Amount
* Treasury Wallet
* Wallet Address entered by the user
* Countdown until expiry
* Payment Status

The wallet entered by the user becomes the wallet used for payment verification.

After successful payment, this wallet automatically becomes the user’s default payout wallet.

⸻

AUTOMATIC PAYMENT VERIFICATION

Implement automatic verification using a Supabase Edge Function.

The Edge Function should periodically inspect pending orders and verify payments on-chain.

Verification must confirm:

* Payment sent from the wallet provided by the user.
* Payment received by the configured treasury wallet.
* Amount received matches the expected SOL amount (within a small tolerance).
* Transaction falls within the order’s validity period.
* Transaction has not already been used.

If verification succeeds:

* Mark the order as Confirmed.
* Store the transaction signature.
* Store the amount received.
* Record the confirmation timestamp.

No manual confirmation flow should exist.

⸻

DEMO CHALLENGE ACTIVATION

Immediately after successful payment:

Create the user’s Demo Challenge.

Initialize:

* Status = Active
* Evaluation Period = 21 Days
* Trading Days = 0
* Win Rate = 0%
* Drawdown = 0%
* Open Positions = 0
* Challenge Progress = 0%

The dashboard should immediately reflect the active challenge.

⸻

REFERRALS

Referral codes are generated during signup.

When a user’s FIRST successful challenge purchase is confirmed:

* Credit 10% of the purchase price to the referrer.
* Ignore subsequent purchases.
* Prevent self-referrals.
* Prevent duplicate commissions.

Referral processing must happen automatically after payment confirmation.
